# Lock

ReentrantLock可以达到synchronized类似的效果：

```java
ReentrantLock lock = new ReentrantLock();
Thread thread1 = new Thread(new Runnable() {
	@Override
	public void run() {
		System.out.println("线程1开始");
		lock.lock();
		System.out.println("线程1锁");
		try {
			Thread.sleep(5000);
		} catch (InterruptedException e) {
			e.printStackTrace();
		}
		System.out.println("线程1结束");
		lock.unlock();
	}
});

Thread thread2 = new Thread(new Runnable() {
	
	@Override
	public void run() {
		System.out.println("线程2开始");
		lock.lock();
		System.out.println("线程2锁");
		try {
			Thread.sleep(5000);
		} catch (InterruptedException e) {
			e.printStackTrace();
		}
		System.out.println("线程2结束");
		lock.unlock();
	}
});

thread1.start();
thread2.start();
```
运行结果：  
```
线程1开始
线程2开始
线程1锁
线程1结束
线程2锁
线程2结束
```
- ReentrantReadWriteLock:
有两个锁，一个是与读相关的锁，称为“共享锁”；另一个是与写有关的锁，称为“排他锁”。就是多个读锁之间不互斥，读锁和写锁互斥，写锁和写锁互斥。

```JAVA
ReentrantReadWriteLock readWriteLock = new ReentrantReadWriteLock();
new Thread() {
	public void run() {
		System.out.println("Thread3 start to write");
		readWriteLock.writeLock().lock();
		System.out.println("Thread3 write locked");
		try {
			Thread.sleep(5000);
		} catch (InterruptedException e) {
			e.printStackTrace();
		}
		readWriteLock.writeLock().unlock();
		System.out.println("Thread3 write over");
	};
}.start();

new Thread() {
	public void run() {
		System.out.println("Thread1 start read");
		readWriteLock.readLock().lock();
		System.out.println("Thread1 read locked");
		try {
			Thread.sleep(5000);
		} catch (InterruptedException e) {
			e.printStackTrace();
		}
		readWriteLock.readLock().unlock();
		System.out.println("Thread1 read over");
	};
}.start();

new Thread() {
	public void run() {
		System.out.println("Thread2 start read");
		readWriteLock.writeLock().lock();;
		System.out.println("Thread2 read locked");
		try {
			Thread.sleep(5000);
		} catch (InterruptedException e) {
			e.printStackTrace();
		}
		readWriteLock.writeLock().unlock();
		System.out.println("Thread2 read over");
	};
}.start();
``` 
运行结果：  
```
Thread3 start to write
Thread3 write locked
Thread1 start read
Thread2 start read
Thread3 write over
Thread2 read locked
Thread2 read over
Thread1 read locked
Thread1 read over

```
